# 快速编辑 (Fast Edit)

## 目标
处理不需要完整设计流程的小型修改、UI微调或明确的 Bug 修复。

## 上下文加载
我们不需要加载所有的设计文档，但必须保持基本的代码规范。
**阅读**：
1.  `@README.md` (项目概览)
2.  `@.agents/reference/common/*.md` (开发标准与最佳实践)

## 执行流程

### 1. 分析与定位
- 分析用户的请求。
- 使用 `go_to_file` 或 `grep_search` 快速定位相关代码文件。

### 2. 评估复杂度 (Safety Check)
- **停止**：如果修改涉及数据库 Schema 变更、复杂的架构重构或新的主要功能。
    - **行动**：告知用户这是一个复杂任务，请使用 `/piv-plan`。
- **继续**：如果是逻辑修复、样式调整或简单的增删改。

### 3. 执行修改
- 直接编辑文件。
- 遵循 Context 中读取到的代码风格规范。

### 4. 验证
- 针对修改的文件运行相关的检查。
    - 类型检查 (例如 `bun run type-check`)。
    - 如果是逻辑修改，尝试运行相关测试（如果有）。

## 输出
简要报告修改了哪些文件以及验证结果。
