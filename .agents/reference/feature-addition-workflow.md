# 功能添加工作流 (Feature Addition Workflow)

本文档描述了在项目中添加新功能时必须遵循的标准工作流程。

## 流程步骤

1. **需求理解与分析**
   - 仔细阅读并深入理解提出的功能需求。
   - 澄清任何模糊或不确定的点。

2. **用户确认需求**
   - 向用户复述或总结你对需求的理解。
   - **必须**获得用户的明确确认，表明理解一致后，方可进入下一步。

3. **数据库设计**
   - *前提条件*：需求已获用户确认。
   - 设计必要的数据模型、表结构、字段类型及关联关系。
   - 将数据库设计方案展示给用户。
   - **必须**获得用户对数据库设计的明确确认，方可进入下一步。

4. **接口设计 (OpenAPI)**
   - *前提条件*：数据库设计已获用户确认。
   - 开始设计后端 API 接口。
   - **必须**使用 `openapi.yml` 格式输出接口定义。
   - 定义应包含完整的路径、HTTP 方法、请求参数及响应结构。
